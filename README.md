# CelestiFrame

CelestiFrame（セレスティフレーム）は、指定した場所・日時における太陽と月の位置を地図上で確認し、風景や建造物と組み合わせた写真の撮影計画を立てるためのPWAです。

## 現在の開発段階

Phase 7（撮影計画の保存）まで実装済みです。

- スマートフォン向け基本画面
- Leaflet / OpenStreetMapの地図
- 撮影地点の選択と現在地取得
- 日付・時刻操作
- 太陽・月の表示切り替え枠
- PWA Manifest / Service Worker / オフライン画面
- 最後に使用した地点と日時の端末保存
- 角度ユーティリティの単体テスト
- 指定地点・日時の太陽方位角と高度角
- 日の出・日の入り
- 地図上の太陽方向線（地平線の下では破線）
- 指定地点・日時の月の方位角と高度角
- 月齢・月相・照度
- 月の出・月の入り
- 地図上の月方向線（地平線の下では破線）
- 端末設定・ライト・ダーク・赤色ナイトのテーマ切り替え
- 選択テーマと日時・地点の端末保存
- 被写体地点の地図指定とドラッグ調整
- 撮影地点から被写体までの距離・方位
- 被写体方向に対する太陽・月の左右差
- 最大93日間の太陽・月の重なり日時検索
- 方位差・高度・月照度・時刻帯・検索間隔の条件指定
- Web Workerによる非同期検索、進捗表示、キャンセル
- 検索候補からメイン日時への反映
- 23時台から0時台への操作で日付を自動繰り越し
- `18:00〜翌06:00`のような深夜をまたぐ検索範囲
- 地名・施設名検索と、検索結果からの撮影地点・被写体設定
- 撮影計画のIndexedDB保存、一覧、復元
- 計画名・メモの編集、複製、削除、お気に入り
- 全計画のJSON書き出し・読み込み
- 地点・日時・天体を復元できる共有URL
- 右上の「◐」と揃えたPWAアイコン

次はPhase 8として被写体高度、標高、焦点距離、画角を使った簡易構図計画を追加します。

## ローカル起動

Service Workerと位置情報を正しく扱うため、ファイルを直接開かずローカルHTTPサーバーを使用してください。

```powershell
npm run serve
```

表示されたローカルURLをブラウザで開きます。

## データとオフライン動作

撮影地点、日時、表示対象などの初期設定は端末内へ保存します。初回オンライン表示後はアプリシェルをキャッシュします。地図タイルは表示済み範囲のみランタイムキャッシュされるため、未表示範囲の地図を完全オフラインで閲覧することはできません。

## 場所検索とNominatim利用上の制限

場所検索にはOpenStreetMap Foundationが運用する公開Nominatim APIを使用します。公開サービスの負荷を抑えるため、入力中の自動補完は行わず、ユーザーが検索を実行した場合だけ通信します。同一検索語の結果は端末内へ7日間保存し、最大1リクエスト/秒を守ります。検索結果にはOpenStreetMapの帰属表示を付けています。

公開Nominatim APIには利用量と用途の制限があり、大規模利用・一括検索・自動補完には使用できません。利用規模が増えた場合は、`js/map/geocoder.js`の検索先をセルフホストまたは別の提供事業者へ切り替えてください。詳細は[Nominatim Usage Policy](https://operations.osmfoundation.org/policies/nominatim/)を確認してください。

## 注意

CelestiFrameが今後表示する天体位置や重なり予測は、写真撮影計画のための計算上の目安です。測量用途には使用できません。重要な撮影では現地確認を行ってください。
