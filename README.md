# CelestiFrame

CelestiFrame（セレスティフレーム）は、指定した場所・日時における太陽・月・星景対象の位置を地図上で確認し、風景や建造物と組み合わせた写真の撮影計画を立てるためのPWAです。

## 現在の開発段階

## 予報雲レイヤー

左上の「予報雲」から、選択中の撮影日時における雲予報を地図の淡いベールとして確認できます。総雲・低層・中層・高層を切り替え、撮影地点の空況指標も併せて見られます。予報はOpen-Meteo経由のGFSグリッド予報値で、前日から16日先までが対象です。

グリッド予報は局地的な雲、地形性の雲、短時間の変化を正確に示すものではありません。最終的な出発判断には、現地の実況・衛星画像・気象警報も確認してください。

## 光害の目安レイヤー

左上の「光害」スイッチから、NASA VIIRSのVNP46A4 2025年次夜間光を日本周辺の地図へ重ねられます。雪のない期間の全観測角度合成を使用し、明るい場所ほど人工光が強い傾向を示します。暗い撮影地を探すための広域的な目安として使えます。

これは空の明るさ、ボートル階級、透明度を直接示すデータではありません。2025年の地表夜間光を使った参考表示であり、地形による遮光、撮影当日の雲・大気、月明かりは反映しません。データ表示には`NASA VIIRS / VNP46A4`の帰属とDOIを付けています。

### VNP46A4タイルの更新

元HDF5はGit管理せず、`data/vnp46a4-source/`へ保存します。NASA Earthdataのダウンロードトークンを一時環境変数`EARTHDATA_TOKEN`へ設定してから、次を実行します。トークンをファイルやシェル履歴へ残さないでください。

```powershell
py -3 -m pip install -r tools/requirements-vnp46a4.txt
py -3 tools/generate-vnp46a4-tiles.py
Remove-Item Env:EARTHDATA_TOKEN
```

対象年・日本周辺を覆うLAADSファイルは`tools/vnp46a4-2025-japan.json`で固定し、生成物は`assets/light-pollution/vnp46a4-2025/`へ出力します。

## Version 1.1の撮影支援

撮影地点候補は、被写体から見た天体方位の反対方向へ1・5・10・20kmの地点を示します。候補地点が道路上か、水上・私有地・立入禁止区域ではないかは自動判定しないため、地図と現地情報を確認してください。

地形断面は撮影地点と被写体の間を21点に分け、国土地理院DEMの標高と地球曲率から見通しを推定します。建物、樹木、造成、積雪、局地的な大気差は含みません。

天の川表示はIAUの銀河座標系に基づく銀河面を10°間隔で計算し、地平線より上のアーチを主表示します。銀河中心が沈んでいる季節・時刻でも、見えているアーチの最高点方位と高度を表示します。表示は幾何学的な方向であり、光害、月明かり、雲、透明度による見えやすさは判定しません。

現地撮影モードは、撮影地点へ到着するまでは撮影地点を案内し、30m以内では被写体方向へ切り替えます。端末方位は磁気センサーの影響を受けるため目安として使用してください。

## データとオフライン動作

撮影地点、日時、表示対象などの初期設定は端末内へ保存します。初回オンライン表示後はアプリシェルをキャッシュします。地図タイルは表示済み範囲のみランタイムキャッシュされるため、未表示範囲の地図を完全オフラインで閲覧することはできません。

## 場所検索とNominatim利用上の制限

場所検索にはOpenStreetMap Foundationが運用する公開Nominatim APIを使用します。公開サービスの負荷を抑えるため、入力中の自動補完は行わず、ユーザーが検索を実行した場合だけ通信します。同一検索語の結果は端末内へ7日間保存し、最大1リクエスト/秒を守ります。検索結果にはOpenStreetMapの帰属表示を付けています。

公開Nominatim APIには利用量と用途の制限があり、大規模利用・一括検索・自動補完には使用できません。利用規模が増えた場合は、`config/runtime-config.json`の`nominatimEndpoint`をセルフホストまたは別の提供事業者へ切り替えてください。アプリ本体の再ビルドは不要です。詳細は[Nominatim Usage Policy](https://operations.osmfoundation.org/policies/nominatim/)を確認してください。

## 外部サービスの実行時設定

場所検索、地図タイル、光害の目安タイル、天気予報の接続先は`config/runtime-config.json`で変更できます。

```json
{
  "nominatimEndpoint": "https://nominatim.openstreetmap.org/search",
  "tileUrl": "https://tile.openstreetmap.org/{z}/{x}/{y}.png",
  "lightPollutionTileUrl": "./assets/light-pollution/vnp46a4-2025/{z}/{x}/{y}.webp",
  "lightPollutionDataYear": 2025,
  "weatherForecastEndpoint": "https://api.open-meteo.com/v1/forecast"
}
```

外部接続先はHTTPSのみ許可します。光害タイルは`assets`配下の同一オリジン相対パスも許可します。タイルURLには`{z}`、`{x}`、`{y}`が必要です。不正な設定や取得失敗時は安全な既定値へ戻ります。OpenStreetMap標準タイルを使う場合は、[Tile Usage Policy](https://operations.osmfoundation.org/policies/tiles/)に従い、URLや帰属表示を維持してください。

## セキュリティとプライバシー

- 入力内容を生成AIやLLMへ送信する処理はありません。検索語は設定された場所検索サービスにだけ送信されます。
- 画面へ表示する検索結果、計画名、メモ、共有URLの被写体名はテキストとして描画し、HTMLとして実行しません。
- 場所検索は120文字、計画JSONは5MBかつ1000件までに制限しています。
- Content Security Policyを設定し、Leafletの外部ファイルはバージョン固定とSubresource Integrityで検証します。
- 撮影地点、被写体地点、日時、計画は端末内へ保存されます。共有URLには地点・日時・被写体名が含まれるため、公開範囲に注意してください。

## 標高データとダイヤモンド判定

地点を確定した時だけ国土地理院の標高APIへ問い合わせ、利用可能な中で最も高精度な数値標高モデルの値を取得します。取得値は地表面の推定標高であり、建物・樹木・積雪を含みません。建造物を狙う場合は「建造物」を選び、地表から照準点までの高さを入力してください。

ダイヤモンド判定は、撮影地点と照準点の標高、距離、地球曲率、標準的な大気差、太陽円盤の角半径を使った撮影計画上の推定です。実際の大気状態、地形による途中遮蔽、雲や視界は判定しません。標高APIの仕様と注意事項は[国土地理院の技術情報](https://maps.gsi.go.jp/development/elevation_s.html)を確認してください。

## 注意

CelestiFrameが今後表示する天体位置や重なり予測は、写真撮影計画のための計算上の目安です。測量用途には使用できません。重要な撮影では現地確認を行ってください。
