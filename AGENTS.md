# CelestiFrame UI QA

UIを変更したCodexは、完了報告・コミット・Pushの前に次を確認する。

- PC幅とモバイル幅（390px前後。狭幅に影響する場合は320pxも）を実画面で確認する。
- ダークとライトの両テーマで、本文・補助文字・対象色の可読性を確認する。
- 天体は単一選択と複数選択を確認し、詳細カードと要約行の順序、間隔、文字色を確認する。
- モバイルのコントロールは、半画面・全画面・最小化・再表示を操作し、各操作の入口が画面から理解できることを確認する。
- スクロール後に最小化しても、再表示ボタンが見えて操作できることを確認する。
- CSSの固定色、テーマ別トークン、後勝ちの詳細セレクターが別テーマを壊していないか、計算済みスタイルでも確認する。
- UI確認で見つけた回帰は、その場の修正だけでなく、可能な範囲で自動テストまたはこのチェック項目へ残す。

## 完了時の公開

- ユーザーがローカルのみを明示しない限り、CelestiFrameの変更はテストと必要な実画面確認後にコミットし、`origin/main` へPushしてから完了報告する。
- スマートフォン実機での確認が必要な変更は、確認可能な状態にするためPushを必須とする。
